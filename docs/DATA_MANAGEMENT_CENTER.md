# Data Management Center

## 버전
HearMe2nite v1.1 STEP1

## 목표
사용자가 자신의 데이터 삭제를 직접 실행하지 않고, 관리자에게 삭제 요청을 보낼 수 있도록 한다.

## 핵심 원칙

- 사용자는 삭제 요청만 가능하다.
- 사용자는 서버 데이터를 직접 삭제할 수 없다.
- 관리자는 요청을 검토하고 승인 또는 보류 메시지를 남긴다.
- 실제 삭제 엔진은 이번 단계에 포함하지 않는다.
- 기존 Room / History / Presence / Firebase 구조는 변경하지 않는다.

## 사용자 앱 기능

### 데이터 관리
앱 상단 사용자 영역의 `데이터 관리` 버튼으로 진입한다.

포함 메뉴:

- 삭제 요청
- 요청 내역
- 데이터 정책

### 삭제 요청
사용자는 삭제 사유를 입력하고 확인 체크 후 요청을 보낸다.

저장 위치:

```text
rooms/{roomCode}/dataManagement/deleteRequests/{requestId}
```

### 요청 내역
사용자는 본인이 보낸 요청만 볼 수 있다.

상태:

```text
pending   검토 중
approved  승인
hold      보류
scheduled 삭제 예정
completed 삭제 완료
canceled  사용자 취소
```

## 관리자 기능
이번 단계에서는 사용자 앱만 구현한다.
관리자 승인/보류 기능은 HearU2nite Platform v1.1에서 구현한다.

## 삭제 엔진
이번 단계에서는 실제 서버 삭제 기능을 넣지 않는다.
실제 삭제는 별도 관리자 승인, 예약 삭제, 운영 로그, 백업 정책을 확정한 뒤 Cloud Function으로 구현한다.

## QA 항목

- [ ] 로그인한 사용자만 데이터 관리 모달을 열 수 있다.
- [ ] Room이 없으면 삭제 요청을 보낼 수 없다.
- [ ] 삭제 사유가 짧으면 요청을 막는다.
- [ ] 확인 체크가 없으면 요청을 막는다.
- [ ] 삭제 요청이 Firebase에 저장된다.
- [ ] 요청 내역에서 내 요청이 보인다.
- [ ] 이미 pending/approved/scheduled 요청이 있으면 중복 요청을 막는다.
- [ ] pending 요청은 취소할 수 있다.
- [ ] 실제 Room 데이터는 삭제되지 않는다.
