# HearMe2nite v1.0 STEP6.2.14.44 종합 검수

검수 기준: 테스트 앱, 로컬 소스, Dom/Sub 권한 계약, 저장·날짜·비동기 계약, PWA·빌드 기준선

안전 범위: 카드 배치 유지, 관리자 앱 미수정, 메인 앱 미배포, Firebase 서버 규칙 미변경

## 통합 개선 우선순위

### 긴급

- 현재 추가 조치가 필요한 긴급 결함은 발견되지 않았다.
- 배포 전에는 Dom/Sub 실제 계정 교차 회귀와 저장 실패·재접속 시나리오를 반드시 다시 실행한다.

### 높음

1. 로컬의 인증 전환, 미디어 업로드 문맥, 날짜·자동 저장, PWA 수정 사항을 하나의 릴리스 후보로 묶어 전체 회귀한다.
2. Dom/Sub 각각 별도 세션에서 권한 전용 영역, 나의 루틴, 관리 카드, 소유자 메모를 검증한다.
3. 배포 후보의 Database 규칙이 테스트 프로젝트 배포 규칙과 일치하는지 읽기 전용 감사 후 확인한다.
4. 저장 중 계정·공간·날짜 전환과 느린 사진 업로드를 실제 네트워크 조건에서 재현한다.

### 보통

1. 긴 설정 화면을 계정·공간·테마·데이터 묶음으로 유지하고 각 하위 화면의 복귀 위치를 회귀 검수한다.
2. 라이트 모드의 경계선·본문·보조 문구 대비를 실제 기기에서 확인한다.
3. PC 앱 프레임과 주변 배경의 연결감을 보강하되 현재 카드 폭·2열 배치는 유지한다.
4. 서비스워커 업데이트 알림 이후 새 버전 적용 절차를 사용자 관점에서 검증한다.

### 낮음

1. 이모지와 선형 아이콘을 장기적으로 한 체계로 통일한다.
2. 카드의 주요 행동·보조 행동·정보 표시를 색상과 높이로 구분한다.
3. 설정 및 도움말 문구의 길이와 어조를 통일한다.

## 역할별 검수

### 1. 20년 차 앱 디자이너

- 잘된 부분: 모바일 카드 폭과 2열 배치가 안정적이며 다크 테마의 감성적 정체성이 분명하다.
- 보강할 부분: PC 프레임이 넓은 배경에서 다소 분리되어 보이고 아이콘 체계가 혼합되어 있다.
- 위험하거나 깨질 가능성이 있는 부분: 카드 구조를 다시 배열하면 익숙한 탐색 흐름과 작은 화면 균형이 동시에 깨질 수 있다.
- 개선 우선순위: 보통 — 구조가 아니라 표면 대비와 아이콘 일관성을 다듬는다.
- 다음 작업 권고사항: 현재 카드 위치·폭·2열 구조를 고정한 채 색상, 테두리, 그림자만 별도 디자인 단계에서 조정한다.

### 2. 20년 차 UI·UX 설계자

- 잘된 부분: 날짜, 요약, 대화, 알림과 여섯 카테고리의 정보 우선순위가 이해하기 쉽다.
- 보강할 부분: 긴 설정 화면의 인지 부담과 일부 관리 전용 표식의 중복 노출을 줄여야 한다.
- 위험하거나 깨질 가능성이 있는 부분: 설정 계층과 뒤로가기를 동시에 바꾸면 사용자가 현재 위치를 잃을 수 있다.
- 개선 우선순위: 높음 — 권한별 보이는 행동과 복귀 경로를 먼저 안정화한다.
- 다음 작업 권고사항: Dom/Sub 태스크별 성공 경로를 짧은 시나리오로 고정해 매 릴리스에서 반복한다.

### 3. 20년 차 앱 프로그래머

- 잘된 부분: 인증 전환 식별자, 오래된 비동기 응답 거부, 이벤트 해제가 명시적으로 보강됐다.
- 보강할 부분: 전역 함수와 화면별 초기화 순서의 결합을 더 줄일 필요가 있다.
- 위험하거나 깨질 가능성이 있는 부분: 로그인 전환 중 이전 계정의 늦은 응답이 새 화면에 반영되는 것이 가장 큰 회귀 위험이다.
- 개선 우선순위: 높음 — 인증·공간·날짜 문맥 검사를 모든 쓰기 흐름에 동일하게 적용한다.
- 다음 작업 권고사항: 저장·업로드·리스너에 공통 요청 문맥 객체와 취소 규약을 사용한다.

### 4. 20년 차 코딩·리팩터링 전문가

- 잘된 부분: CSS가 순서 보존 레이어로 분리됐고 핵심 계약 검사가 리팩터링 경계를 보호한다.
- 보강할 부분: 큰 화면 모듈과 중복 DOM 제어 코드를 기능 단위로 더 축소할 수 있다.
- 위험하거나 깨질 가능성이 있는 부분: 대규모 파일 이동과 기능 변경을 같은 단계에서 하면 원인 추적이 어려워진다.
- 개선 우선순위: 보통 — 동작을 고정한 뒤 작은 단위로 분리한다.
- 다음 작업 권고사항: 한 번에 한 모듈만 정리하고 매 단계에서 56개 핵심 계약과 앱 기준선을 실행한다.

### 5. 앱 구조·아키텍처 설계자

- 잘된 부분: 접근 정책, 저장 상태, UI 이벤트, 화면 탐색 책임이 분리되는 방향이 적절하다.
- 보강할 부분: 인증 상태, Room 상태, 선택 날짜를 단일 세션 문맥으로 관리할 필요가 있다.
- 위험하거나 깨질 가능성이 있는 부분: 화면마다 권한 판단을 독자 구현하면 Dom/Sub 규칙이 서서히 달라질 수 있다.
- 개선 우선순위: 높음 — 권한과 비동기 문맥을 중앙 계약으로 유지한다.
- 다음 작업 권고사항: UI 권한 표시와 Database 규칙의 대응표를 릴리스 체크리스트에 포함한다.

### 6. 모바일·Fold·태블릿·PC 반응형 검수자

- 잘된 부분: 360px 모바일의 여섯 화면, 673px Fold, 768px 태블릿, 1440px PC에서 가로 넘침이 없었다.
- 보강할 부분: PC 프레임의 배경 연결감과 일부 긴 설정 화면의 세로 피로도를 개선할 수 있다.
- 위험하거나 깨질 가능성이 있는 부분: 카드 열 수를 늘리면 현재 모바일·태블릿의 시선 흐름이 깨질 수 있다.
- 개선 우선순위: 보통 — 현재 최대 폭과 2열 카드 계약을 유지한다.
- 다음 작업 권고사항: 360/673/768/1440 폭을 시각 회귀 기준으로 고정하고 라이트·다크를 각각 캡처한다.

### 7. Dom/Sub 권한 및 개인정보 검수자

- 잘된 부분: 관리 카드, 소유자 메모, Sub 루틴의 읽기·쓰기 역할이 핵심 계약으로 명문화됐다.
- 보강할 부분: 동일 브라우저 계정 전환과 별도 브라우저 동시 접속을 모두 검증해야 한다.
- 위험하거나 깨질 가능성이 있는 부분: UI 숨김만 믿거나 이전 리스너가 남으면 권한 정보가 잘못 노출될 수 있다.
- 개선 우선순위: 높음 — 서버 규칙과 UI 표시를 함께 검증한다.
- 다음 작업 권고사항: Dom×Sub 권한 행렬의 허용·거부 항목을 에뮬레이터 및 실제 테스트 계정에서 반복한다.

### 8. 저장·날짜·비동기 처리 검수자

- 잘된 부분: KST 날짜, 불변 저장 문맥, 완료 시점 재검사, 사진 업로드 정리가 보강됐다.
- 보강할 부분: 오프라인·느린 연결·날짜 자정·대용량 사진 조건의 실사용 시험이 필요하다.
- 위험하거나 깨질 가능성이 있는 부분: 업로드 중 계정·Room·날짜 전환 시 이전 결과나 파일이 남는 상황이다.
- 개선 우선순위: 높음 — 실패와 전환 시나리오를 우선 시험한다.
- 다음 작업 권고사항: 저장 시작→문맥 전환→완료 순서를 자동화하고 UI·Database·Storage 결과를 함께 확인한다.

### 9. 테스트·배포·회귀 검수자

- 잘된 부분: 56개 핵심 계약과 앱 기준선이 통과했고 테스트 앱의 콘솔 경고·오류는 없었다.
- 보강할 부분: 라이트 모드와 별도 Sub 세션의 운영 앱 시각 회귀를 추가해야 한다.
- 위험하거나 깨질 가능성이 있는 부분: 로컬 수정과 운영 테스트 앱이 다른 상태에서 결과를 혼동할 수 있다.
- 개선 우선순위: 높음 — 배포 후보 식별과 검증 대상을 명확히 분리한다.
- 다음 작업 권고사항: 빌드→로컬 회귀→규칙 읽기 감사→승인→테스트 Hosting 배포 순서를 체크리스트로 고정한다.

### 10. 실제 사용자 관점의 최종 검수자

- 잘된 부분: 홈에서 원하는 기록으로 이동하기 쉽고 모든 카테고리에서 홈 복귀가 명확하다.
- 보강할 부분: 관리 역할 용어와 일부 설정 설명을 더 일상적인 언어로 다듬을 수 있다.
- 위험하거나 깨질 가능성이 있는 부분: 저장 여부가 불분명하거나 계정 전환 뒤 이전 화면이 남으면 신뢰가 크게 떨어진다.
- 개선 우선순위: 높음 — 데이터가 정확히 저장되고 역할에 맞게 보인다는 확신을 우선한다.
- 다음 작업 권고사항: 신규 Dom, 신규 Sub, 기존 장기 사용자의 하루 기록 과정을 각각 10분 내외로 관찰한다.

## 이번 회귀 결과

- 운영 테스트 앱 여섯 카테고리 모두 진입 및 홈 복귀 성공
- 360px 모바일 전 화면 가로 넘침 없음
- 673px Fold, 768px 태블릿, 1440px PC의 홈·설정 화면 가로 넘침 및 화면 밖 요소 없음
- 운영 테스트 앱 콘솔 warning/error 0건
- 로컬 핵심 계약 56개 통과, 앱 기준선 통과
- 운영 테스트 앱에는 로컬에서 제거한 `관리 전용` 중복 접근성 문구가 아직 남아 있음: 미배포 상태이므로 예상된 차이

## 배포 전 중단 조건

- Dom 또는 Sub에서 허용되지 않은 데이터가 보이거나 저장되는 경우
- 로그인·Room·날짜 전환 후 이전 문맥의 저장 완료가 새 화면에 반영되는 경우
- Database 배포 규칙과 로컬 규칙이 달라진 경우
- 모바일 또는 Fold에서 가로 스크롤이 생기는 경우
- 서비스워커 갱신이 작성 중 화면을 강제로 새로고침하는 경우

## 로컬 배포 후보 최종 검증 결과

- 로컬 최신본 Sub 세션에서 `주인의 메시지` 노출 및 관리 전용 카드 비노출 확인
- Dom 전용 영역이 `hidden`, `aria-hidden`, `inert`로 함께 차단되는 것 확인
- Sub의 `나의 루틴` 화면 정상 노출 확인
- 라벤더 라이트·다크 미리보기 모두 가로 넘침 없음
- 테마 설정을 저장하지 않고 기존 라이트 모드로 복구
- 테스트 프로젝트 배포 규칙과 로컬 `database.rules.json` 일치 확인
- 핵심 계약 56개 및 앱 기준선 통과
- 실제 사진 업로드·저장 실패 조작은 테스트 데이터를 만들 수 있어 이번 읽기 중심 검증에서는 수행하지 않음
