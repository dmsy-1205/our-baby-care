# UI Guideline

## 목표

HearMe2nite는 기능이 많은 앱보다 `매일 열고 싶은 앱`을 목표로 합니다.

## 디자인 원칙

- 부드러운 카드형 UI
- 통일된 Radius
- 과하지 않은 Shadow
- 모바일 우선 Responsive
- 팝업 중심 입력
- 불필요한 화면 이동 최소화
- 따뜻하고 조용한 감성 유지

## Header

- HearMe2nite 로고와 제목 유지
- `❓` Help Center 버튼 유지
- Presence Indicator는 오른쪽 상단에서 작게 표시

## Card

- 주요 기능은 카드 단위로 표현
- 누르면 팝업이 열리는 구조 유지
- 카드 크기는 모바일에서 너무 길어지지 않게 관리

## Popup

- 모바일, Fold, Tablet, PC에서 중앙 정렬
- 닫기 버튼 명확히 제공
- 입력 후 `저장하고 닫기` 흐름 유지

## Anniversary

- 아이콘 카드는 고정하지 않는다.
- 사용자가 어떤 기념일을 등록할지 모르기 때문에 목록은 다가오는 순으로 표시한다.
- 카드 최대 3개 + 더보기 구조를 목표로 한다.

## History

- Calendar 중심
- 날짜 클릭 → 하루 기록 팝업
- 사진 모아보기는 별도 영역으로 정리

## Tone

문구는 명령형보다 안내형을 사용합니다.

예: `입력하세요`보다 `기록해 주세요`를 우선합니다.
