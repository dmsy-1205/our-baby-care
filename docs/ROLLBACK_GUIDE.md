# HearMe2nite 배포 및 롤백 기준

## 기준점

배포 직전 GitHub Desktop에서 모든 변경 파일을 하나의 커밋으로 저장합니다. 이 커밋 이름과 앱의 STEP 버전이 같아야 합니다.

- 현재 기준 버전: `v1.0 STEP6.2.13.8.9`
- 권장 태그: `hearme2nite-main-v1.0-step6.2.13.8.9`
- 공개 배포 파일은 `node scripts/build-public.js`로 다시 생성합니다.

## 권장 배포 순서

1. GitHub Desktop에서 현재 변경을 커밋하고 원격 저장소에 올립니다.
2. 테스트 프로젝트 `hearme2nite1205`에 먼저 배포합니다.
3. 로그인, 도움말, 기록 저장, 문의 보내기를 확인합니다.
4. 문제가 없을 때 메인 프로젝트 `our-baby-care`에 같은 커밋을 배포합니다.

## 문제가 생겼을 때

가장 빠른 복구는 Firebase Console의 Hosting 릴리스 기록에서 직전 정상 버전을 다시 게시하는 것입니다. 코드까지 되돌려야 할 때는 GitHub Desktop에서 직전 정상 커밋으로 새 복구 커밋을 만든 뒤 두 프로젝트에 다시 배포합니다.

데이터베이스와 Storage 규칙은 Hosting과 별도로 관리합니다. 화면 문제를 복구할 때 규칙까지 함께 되돌리지 않습니다.
