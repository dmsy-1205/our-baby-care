# Dom/Sub 기능 권한 기준

이 문서는 화면 표시와 실제 데이터 작업이 따라야 하는 제품 기준이다. `숨김`은 버튼만 숨기는 것이 아니라 Sub가 기능의 존재와 데이터 내용을 알 수 없어야 한다는 뜻이다.

| 기능 | Dom | Sub | 표시 원칙 |
|---|---|---|---|
| 우리의 대화 | 읽기·작성 | 읽기·작성 | 양쪽 모두 표시 |
| 오늘의 약속 | 생성·수정·삭제·확인 | 확인·수행 상태 입력 | 역할에 맞는 동작만 표시 |
| 나의 루틴 | 확인 | 생성·수정·기록 | Sub가 관리하는 개인 루틴 |
| 오늘의 컨디션 | 확인·코멘트 | 작성·수정 | 기록은 Sub 중심 |
| 오늘의 기록 | 확인·코멘트 | 작성·수정 | 기록은 Sub 중심 |
| 주인의 피드백 | 작성·수정 | 읽기 전용 | Sub에게 작성 선택지는 숨김 |
| 오늘의 선물 | 작성·수정 | 읽기 전용 | Sub에게 내용만 표시 |
| Dom 비공개 메모 | 읽기·작성·수정 | 접근 불가 | Sub에게 존재 자체를 숨김 |
| 우리의 기록·통계 | Room 범위 읽기 | Room 범위 읽기 | 권한에 맞는 민감 정보 제외 |
| 내 계정·테마 | 본인 정보 관리 | 본인 정보 관리 | 각자 자신의 설정만 변경 |
| 우리의 공간 | 연결·초대 관리 | 연결 상태 확인·허용된 참여 작업 | 위험 작업은 역할 재확인 |
| 관리자 센터 | 관리자 계정만 | 관리자 계정만 | 일반 Dom/Sub 역할과 분리 |

## 자동 검수에 필요한 핵심 조건

- Sub 화면에는 Dom 비공개 메모의 제목, 빈 상태, 개수, 알림이 모두 없어야 한다.
- Sub의 피드백·선물 화면에는 Dom용 작성 선택지나 편집 입력란이 없어야 한다.
- Dom과 Sub의 코멘트는 원본 기록 수정 권한과 분리한다.
- Room 또는 계정 전환 직후 이전 사용자의 권한 상태가 남지 않아야 한다.
- 관리자 여부는 Dom 역할이 아니라 별도의 관리자 권한 값으로 확인한다.
