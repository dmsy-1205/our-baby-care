# 저장·날짜·임시 데이터 기준

## 공통 상태

| 상태 | 의미 | 화면 표시 |
|---|---|---|
| 작성 중 | 사용자가 값을 바꾸는 중 | 방해하지 않는 작은 표시 |
| 저장 대기 | 명시적 저장이 필요한 사진이 선택됨 | 저장할 장수와 저장 버튼 표시 |
| 저장 중 | 서버 전송 진행 중 | 중복 조작 방지 |
| 저장 완료 | 서버 확인까지 끝남 | 완료 시각 또는 짧은 완료 표시 |
| 저장 실패 | 서버 확인 실패 | 재시도와 원인 안내 |

## 자동 저장 항목

텍스트, 선택값, 수분, 상태값처럼 되돌리기 쉬운 작은 데이터는 자동 저장한다. 입력 직후에는 `작성 중`, 서버 확인 전에는 `저장 중`, 확인 후에는 `저장 완료`로 표시한다.

## 명시적 저장 항목

오늘의 순간과 식사 사진처럼 용량이 크고 선택 취소가 필요한 데이터는 `저장 대기`를 거친다. 같은 계정·Room·날짜 안에서 화면을 나갔다 돌아오면 대기를 유지할 수 있다. 다음 경우에는 그대로 넘기지 않는다.

- 로그아웃 또는 계정 전환
- Room 전환
- 기록 날짜 전환
- 업로드 완료 후 서버 확인

전환 시 저장 대기가 있으면 저장 또는 취소를 선택하게 하고, 다른 계정·Room·날짜에 임시 사진이 나타나지 않게 한다.

## 날짜 변경 순서

1. 현재 날짜의 자동 저장 대기 작업을 마무리한다.
2. 명시적 저장 대기 사진이 있으면 사용자에게 알린다.
3. 새 날짜 데이터를 불러온다.
4. 화면과 코멘트·알림·통계의 날짜 키를 함께 바꾼다.
5. 이전 날짜의 비동기 응답이 새 날짜 화면을 덮지 못하게 요청 식별자를 확인한다.
