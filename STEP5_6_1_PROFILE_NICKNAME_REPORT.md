# HearMe2nite STEP5.6.1 프로필 닉네임 및 채팅 이름 고정 보고서

## 구현 내용
- 사용자별 닉네임 저장: `users/{uid}/profile/nickname`
- 프로필 화면에서 닉네임 설정 및 변경
- 채팅 이름 입력란 읽기 전용 고정
- 새 채팅 메시지에 현재 닉네임을 `sender`로 저장
- 기존 메시지의 발신자 이름은 변경하지 않음
- 닉네임 미설정 시 이메일 앞부분을 안전한 기본 이름으로 사용

## 보호 범위
- 기존 UID, Room, roomMembers, Invite, Presence 변경 없음
- 기존 채팅 메시지 마이그레이션 없음
- 기존 사용자에게 닉네임 설정을 강제하지 않음
- Firebase Rules의 자기 사용자 경로 쓰기 정책을 그대로 사용

## 검수 항목
1. 기존 사용자 로그인 및 Room 복구
2. 내 프로필 열기
3. 2~20자 닉네임 저장
4. 새 채팅 전송 후 sender 값 확인
5. 새로고침 후 닉네임 유지 확인
6. 닉네임 변경 후 새 메시지만 변경된 이름으로 표시되는지 확인
7. 상대방 화면에서 닉네임 표시 확인
