## 0. Compact Account Bar

- [ ] 1행 왼쪽에 닉네임이 표시된다.

- [ ] 모바일, Fold, 태블릿, PC에서 계정 영역이 2줄 구조를 유지한다.
- [ ] 계정 영역 변경이 Auth, Room, Profile 저장 구조에 영향을 주지 않는다.

## 1. 기존 사용자 회귀 테스트

- [ ] 기존 사용자가 이메일 인증 요구 없이 로그인된다.
- [ ] 기존 `users/{uid}/activeRoom`이 정상 복구된다.
- [ ] 기존 Room의 오늘 기록과 기록실이 표시된다.
- [ ] 상대방 Presence와 마지막 접속 정보가 표시된다.
- [ ] 로그아웃 후 다시 로그인해도 동일 Room이 복구된다.

## 2. 신규 회원 이메일 인증

- [ ] 신규 이메일로 회원가입이 된다.
- [ ] 인증 전에는 앱 본문과 Room 기능에 진입할 수 없다.
- [ ] 인증 안내에 스팸함·정크메일함 확인 문구가 표시된다.
- [ ] 인증 메일 다시 보내기는 60초 제한이 적용된다.
- [ ] 이메일 링크 인증 후 `인증 완료 확인`으로 앱에 진입한다.
- [ ] 신규 사용자에게만 `emailVerificationRequired: true`가 기록된다.
- [ ] 기존 사용자에게 인증 정책 필드를 일괄 생성하지 않는다.

## 3. Room과 초대

- [ ] 새 Room 생성이 정상이다.
- [ ] Room 생성자의 `owner` 권한과 relationshipRole이 저장된다.
- [ ] 초대코드가 생성된다.
- [ ] 다른 신규 사용자가 초대코드로 참여한다.
- [ ] Owner와 Partner가 동일 Room에 연결된다.
- [ ] 사용 완료된 초대코드가 재사용되지 않는다.
- [ ] 다른 Room 데이터가 화면에 노출되지 않는다.

## 4. 기록 기능

- [ ] 오늘의 약속이 표시되고 기록된다.
- [ ] 기분, 기상시간, 운동, 체중, 수분이 저장된다.
- [ ] 오늘의 기록과 사진이 저장된다.
- [ ] 관리(Dom) 전용 필드는 기록(Sub)에서 수정할 수 없다.
- [ ] 날짜 변경 후 해당 날짜의 데이터가 정상 로드된다.
- [ ] 기록실 캘린더와 하루 기록 팝업이 정상이다.
- [ ] 채팅 송수신과 시간이 정상 표시된다.

## 5. 사용설명서

- [ ] 사용 가이드가 열린다.
- [ ] `오늘의 약속`이 핵심 기능으로 표시된다.
- [ ] 공간 연결 → 약속 확인 → 하루 기록 흐름이 보인다.
- [ ] 각 카드와 검색 기능이 정상 동작한다.
- [ ] 닫기 버튼과 `확인했어요` 버튼이 정상 동작한다.

## 6. 연결과 오류 처리

- [ ] 인터넷 연결 해제 시 오프라인 안내가 표시된다.
- [ ] 재연결 후 Firebase 연결 상태가 복구된다.
- [ ] 빠른 연속 클릭으로 저장·초대 요청이 중복 실행되지 않는다.
- [ ] Firebase permission 오류가 사용자 데이터 초기화로 이어지지 않는다.
- [ ] 브라우저 새로고침 후 로그인과 Room 상태가 복구된다.

## STEP5.6.1.5 데이터 관리 추가 QA
- [ ] 데이터 관리 모달이 정상적으로 열린다.
- [ ] 계정 삭제 / Room 연결 해제 / Room 전체 삭제 중 하나를 선택할 수 있다.
- [ ] Room 전체 삭제 선택 시 공동 데이터 경고가 표시된다.
- [ ] 사유 10자 미만 또는 확인 미체크 시 요청이 차단된다.
- [ ] 요청 저장 후 요청 종류와 상태가 요청 내역에 표시된다.
- [ ] 진행 중인 요청이 있으면 중복 요청 버튼이 비활성화된다.
- [ ] pending/reviewing/hold 요청은 취소할 수 있다.
- [ ] 기존 요청에 requestType이 없어도 “기존 삭제 요청”으로 표시된다.
- [ ] 실제 Room/Auth/Storage 데이터는 삭제되지 않는다.
